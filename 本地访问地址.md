# 本地开发访问地址

## 推荐：后端 Docker + 前端本地

```bash
# 1. 只启动后端与依赖（不启动 Docker 前端）
docker compose up -d db backend adminer mailcatcher proxy

# 2. 本地启动前端（热更新）
cd frontend
bun install   # 首次
bun run dev
```

| 服务 | 地址 |
|------|------|
| **前端（本地 Vite）** | http://localhost:5173 |
| 后端 API | http://localhost:8000 |
| API 文档 (Swagger) | http://localhost:8000/docs |
| Adminer | http://localhost:8080 |
| Mailcatcher | http://localhost:1080 |
| Traefik 面板 | http://localhost:8090 |

本地开发时，Vite 会把 `/api` 代理到 `http://localhost:8000`（不要用 `127.0.0.1`，Windows 上常被 LODOP/CLodop 打印控件占用 8000 端口），无需额外配置 CORS。

若曾启动过 Docker 前端，请先执行：`docker compose stop frontend`（避免 5173 端口冲突）。

### 登录账号（本地默认）

在 http://localhost:5173/login 使用项目根目录 `.env` 中的超级用户（首次启动后端时会自动创建）：

| 项 | 值 |
|----|-----|
| 邮箱 | `admin@example.com` |
| 密码 | `changethis` |

若你修改过 `.env` 里的 `FIRST_SUPERUSER` / `FIRST_SUPERUSER_PASSWORD`，以 `.env` 为准。

## 可选：全套 Docker

```bash
docker compose watch
```

登录账号同上（见 `.env` 中的 `FIRST_SUPERUSER` / `FIRST_SUPERUSER_PASSWORD`）。
